# 通用BI分析知识库

## 核心概念

### 什么是通用BI分析
通用BI（Business Intelligence）分析是一个综合性的数据分析场景，支持多种数据查询和分析需求，包括：
- 数据探索和发现
- 多维度交叉分析
- 灵活的自定义查询
- 知识库检索
- 基于React模式的智能SQL生成

## 分析模式

### 1. 模板模式
当用户的问题命中预定义的QA模板时，系统会：
1. 识别用户问题对应的模板
2. 引导用户完善模板参数
3. 基于模板和参数生成SQL
4. 执行查询并返回结果

**优点**：
- 快速响应
- SQL质量高
- 参数验证严格

**适用场景**：
- 常见的标准查询
- 结构化程度高的问题
- 需要精确控制参数

### 2. React模式
当用户问题未命中模板时，系统会采用React模式：
1. 使用大模型理解用户意图
2. 通过多轮对话逐步明确需求
3. 基于schema智能生成SQL
4. 执行查询并提供结果

**优点**：
- 灵活性高
- 支持复杂查询
- 自然语言交互友好

**适用场景**：
- 复杂的查询需求
- 非标准问题
- 需要多轮对话的场景

### 3. QA模式
用户直接点击右侧QA模板库中的具体问题：
1. 显示模板详情和参数定义
2. 引导用户填写参数
3. 生成SQL并执行查询

**优点**：
- 操作简单
- 问题明确
- 参数填写有引导

**适用场景**：
- 新手用户
- 快速查询
- 参数化查询

## 数据分析能力

### 基础查询
- **点查**：查询单个时间点的指标值
- **明细查询**：查询详细数据记录
- **聚合查询**：按维度分组聚合

### 时序分析
- **趋势分析**：指标随时间的变化趋势
- **同环比**：同比和环比增长率
- **累积分析**：累计值计算

### 维度探索
- **维度拆解**：多维度交叉分析
- **TopN排名**：排名分析
- **维度对比**：维度间对比分析

### 智能分析
- **异常检测**：数据异常识别
- **智能下钻**：自动根因分析
- **自然语言筛选**：语义化条件筛选

## 使用指南

### 场景选择
当需要综合性数据分析时，选择"通用BI分析"场景，该场景提供：
- 最丰富的分析能力
- 最灵活的查询方式
- 最全面的模板库

### 模式切换
三种模式可以随时切换：
1. **自动模式**（默认）：系统自动判断使用模板模式还是React模式
2. **模板模式**：强制使用模板模式，适合快速标准查询
3. **React模式**：强制使用React模式，适合复杂自定义查询

### 提问技巧
- **明确时间范围**：例如"最近30天"、"2024年1月"
- **指定指标**：例如"销售额"、"用户数"、"转化率"
- **说明维度**：例如"按产品类别"、"按渠道"、"按地区"
- **提供上下文**：如果需要多轮对话，先说明背景和目标

## 常见问题类型

### 销售类
- 查询销售额、订单量等指标
- 分析产品、渠道、地区销售表现
- 同比、环比增长分析

### 用户类
- 查询活跃用户数、新增用户数
- 分析用户留存、转化、行为路径
- 用户分群和画像分析

### 运营类
- 分析活动效果
- 监控关键指标趋势
- 异常检测和预警

## 最佳实践

1. **从小开始**：先从简单查询开始，逐步深入了解数据
2. **善用模板**：优先使用预定义模板，提高效率
3. **参数探索**：尝试不同的参数组合，发现更多洞察
4. **多维度分析**：从不同维度交叉分析，获得全面视角
5. **关注异常**：留意异常数据，可能发现业务问题或机会

## 注意事项

1. **数据权限**：确保有权限访问所需数据
2. **性能考虑**：大范围或复杂查询可能需要较长时间
3. **结果解读**：理解数据背后的业务含义
4. **数据质量**：注意数据完整性和准确性
5. **隐私保护**：遵守数据安全和隐私规范
