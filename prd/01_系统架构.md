# ChatBI 对话式数据分析平台 - 系统架构

## 1. 项目概述

ChatBI 是一个多用户对话式数据查询和分析平台，基于nanobot的agent核心能力构建，支持自然语言交互进行数据查询、分析和洞察。

**核心价值**：
- 通过对话式交互降低数据分析门槛
- 多数据源统一查询（RAG知识库、SQL数据库）
- 安全隔离的多租户架构
- 灵活的扩展能力（自定义skill）

### 2.2 核心组件说明

| 组件 | 说明 | 技术选型 |
|------|------|----------|
| **前端** | Vue/React单页应用 | Vue3 + TypeScript + Element Plus |
| **API网关** | RESTful API服务 | FastAPI + Uvicorn |
| **Agent引擎** | 消息处理和工具调用核心 | AsyncIO + Tool Registry |
| **Loop队列** | 用户请求异步处理队列 | Redis / Python Queue |
| **Session管理** | 用户会话生命周期管理 | SQLite / JSON |
| **Channel适配器** | 钉钉/飞书/自建前端适配 | 多通道接口 |
| **Python沙箱** | 安全执行Python代码 | opensandbox |
| **NAS存储** | 用户级文件隔离存储 | 本地文件系统 |

### 2.3 技术栈

- **后端**：Python 3.10+, FastAPI, Pydantic, SQLAlchemy, Redis
- **前端**：Vue3, TypeScript, Element Plus, Axios, Pinia
- **AI/ML**：OpenAI/LiteLLM, RAG服务
- **存储**：SQLite, JSON文件, 本地文件系统
- **安全**：Opensandbox (Python沙箱)

## 4. 安全架构

### 4.1 安全机制

```
用户请求
  │
  ▼
┌──────────────────┐
│ 身份认证         │  ← JWT Token
│ 通道验证         │  ← Channel Token
└──────────────────┘
  │
  ▼
┌──────────────────┐
│ 请求限流         │  ← 用户级限速
│ 参数校验         │  ← Pydantic
└──────────────────┘
  │
  ▼
┌──────────────────┐
│ 沙箱隔离         │  ← Opensandbox
│ 资源限制         │  ← CPU/Memory/Disk
└──────────────────┘
  │
  ▼
┌──────────────────┐
│ 权限控制         │  ← 文件系统隔离
│ 审计日志         │  ← 操作记录
└──────────────────┘
```

### 4.2 数据隔离

- **用户隔离**：workspace/{user_id}_{channel}/
- **会话隔离**：sessions/{user_id}_{channel}/{conversation_id}.json
- **沙箱隔离**：每个请求独立沙箱环境
- **资源限制**：CPU、内存、磁盘配额

## 5. 监控与运维

### 5.1 监控指标

- **系统指标**：CPU、内存、磁盘、网络
- **业务指标**：活跃用户数、请求QPS、平均响应时间
- **LLM指标**：Token消耗、调用成功率、响应时间
- **工具指标**：工具调用次数、成功率、执行时长

### 5.2 日志管理

- **访问日志**：请求路径、用户、响应时间
- **业务日志**：对话内容、工具调用、执行结果
- **错误日志**：异常堆栈、错误上下文
- **审计日志**：用户操作、数据访问
