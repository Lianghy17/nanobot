# ChatBI 详细需求规格说明书

## 1. 功能性需求

### 1.1 用户管理

#### 1.1.1 用户身份识别
- **需求编号**: FR-USER-001
- **需求描述**: 系统需要识别不同用户的身份，支持多用户并发访问
- **实现方式**: 
  - 自建前端：使用JWT Token识别用户ID
  - 钉钉/飞书：使用企业ID + 用户ID组合识别
- **用户标识格式**: `{channel}:{user_id}`
  - 自建: `web:user_123`
  - 钉钉: `dingtalk:ding_abc`
  - 飞书: `feishu:fei_xyz`

#### 1.1.2 用户会话隔离
- **需求编号**: FR-USER-002
- **需求描述**: 不同用户的数据和操作必须完全隔离
- **实现方式**:
  - 文件存储: `/workspace/{user_channel}/`
  - 会话存储: `/workspace/sessions/{user_channel}/`
  - 沙箱环境: 每个用户请求独立沙箱

### 1.2 对话管理

#### 1.2.1 新建对话
- **需求编号**: FR-CONV-001
- **需求描述**: 用户可以创建新的对话会话
- **流程**:
  1. 用户点击"新建对话"按钮
  2. 前端弹出场景选择对话框
  3. 用户选择场景编码和场景名称
  4. 后端生成唯一的`conversation_id`
  5. 创建会话文件: `/workspace/sessions/{user_channel}/{conversation_id}.json`
  6. 前端跳转到对话页面
- **conversation_id生成规则**: `conv_{timestamp}_{random}`
  - 示例: `conv_20240315_123456_abc123`

#### 1.2.2 持续对话
- **需求编号**: FR-CONV-002
- **需求描述**: 在同一对话中保持上下文连续性
- **规则**:
  - 对话创建后，`conversation_id`保持不变
  - 所有消息追加到同一会话文件
  - 支持上下文引用（最大历史消息数可配置）
  - 支持会话暂停和恢复

#### 1.2.3 对话历史管理
- **需求编号**: FR-CONV-003
- **需求描述**: 支持查看和管理历史对话
- **功能**:
  - 列表展示历史对话（时间倒序）
  - 支持按时间、场景筛选
  - 支持删除对话
  - 支持导出对话记录

### 1.3 场景管理

#### 1.3.1 场景配置
- **需求编号**: FR-SCENE-001
- **需求描述**: 基于配置文件管理不同业务场景
- **配置文件格式**:
```json
{
  "scenes": [
    {
      "scene_code": "sales_analysis",
      "scene_name": "销售数据分析",
      "description": "用于分析销售业绩、趋势、产品表现等",
      "supported_skills": ["sql_query", "file_read", "python_execute"],
      "default_model": "gpt-4",
      "max_iterations": 15
    },
    {
      "scene_code": "user_behavior",
      "scene_name": "用户行为分析",
      "description": "分析用户留存、活跃、转化等行为数据",
      "supported_skills": ["sql_query", "rag_search", "python_execute"],
      "default_model": "gpt-4",
      "max_iterations": 20
    }
  ]
}
```

#### 1.3.2 场景切换
- **需求编号**: FR-SCENE-002
- **需求描述**: 支持在不同场景中切换
- **规则**:
  - 新建对话时选择场景
  - 场景中不可切换（必须新建对话）
  - 每个场景可配置不同的技能和参数

### 1.4 核心能力（Skills）

#### 1.4.1 文件读取
- **需求编号**: FR-SKILL-001
- **需求描述**: 读取用户上传的数据文件
- **支持格式**: CSV, Excel, JSON, Parquet, TXT
- **限制**:
  - 文件大小: 最大 50MB
  - 读取范围: 仅限用户目录 `/workspace/{user_channel}/`
- **工具名称**: `read_file`
- **参数**:
```json
{
  "file_path": "string",  // 相对路径，如: "data/sales.csv"
  "limit": "integer"      // 可选，读取行数限制
}
```

#### 1.4.2 文件查询
- **需求编号**: FR-SKILL-002
- **需求描述**: 在文件中搜索特定内容
- **工具名称**: `search_file`
- **参数**:
```json
{
  "file_path": "string",
  "pattern": "string",     // 搜索模式（支持正则）
  "case_sensitive": "boolean"
}
```

#### 1.4.3 Python代码编写
- **需求编号**: FR-SKILL-003
- **需求描述**: 生成Python代码文件
- **工具名称**: `write_python`
- **参数**:
```json
{
  "file_path": "string",   // 保存路径，如: "analysis/sales_plot.py"
  "code": "string",        // Python代码内容
  "description": "string"  // 代码说明
}
```
- **限制**:
  - 禁止写入系统目录
  - 必须包含注释说明
  - 建议包含main函数或执行入口

#### 1.4.4 Python代码执行
- **需求编号**: FR-SKILL-004
- **需求描述**: 在沙箱环境中执行Python代码
- **工具名称**: `execute_python`
- **参数**:
```json
{
  "file_path": "string",   // 要执行的Python文件
  "timeout": "integer",    // 超时时间（秒），默认60s
  "args": "array"          // 命令行参数
}
```
- **安全限制**:
  - 使用opensandbox创建独立沙箱
  - 禁止网络访问（白名单模式）
  - 限制CPU: 1核
  - 限制内存: 512MB
  - 限制磁盘: 1GB
  - 超时自动终止

#### 1.4.5 RAG知识库查询
- **需求编号**: FR-SKILL-005
- **需求描述**: 查询RAG知识库获取文档、表结构、QA示例
- **工具名称**: `rag_search`
- **API接口**: `http://my_rag_v1`
- **参数**:
```json
{
  "scene_code": "string",  // 场景编码
  "query": "string",       // 用户问题
  "type": "string"         // 可选: doc(文档), schema(表结构), qa(QA示例), all(全部)
}
```
- **返回示例**:
```json
{
  "docs": [
    {
      "title": "销售数据表说明",
      "content": "sales表包含...",
      "score": 0.95
    }
  ],
  "schemas": [
    {
      "table_name": "sales",
      "columns": [...],
      "description": "销售主表"
    }
  ],
  "qa_examples": [
    {
      "question": "查询上个月的销售额",
      "sql": "SELECT SUM(amount) FROM sales WHERE date >= ..."
    }
  ]
}
```

#### 1.4.6 SQL执行
- **需求编号**: FR-SKILL-006
- **需求描述**: 执行SQL查询语句
- **工具名称**: `execute_sql`
- **API接口**: `http://my_hive_sql_exec`
- **参数**:
```json
{
  "sql_text": "string",    // SQL语句
  "scene_code": "string"   // 场景编码
}
```
- **安全限制**:
  - 禁止DDL（CREATE/DROP/ALTER）
  - 禁止DML（INSERT/UPDATE/DELETE）
  - 仅允许SELECT查询
  - 自动添加LIMIT 10000限制
  - 超时时间: 30秒
- **返回格式**:
```json
{
  "success": true,
  "data": [...],
  "columns": [...],
  "row_count": 100,
  "execution_time_ms": 1234
}
```

### 1.5 Channel管理

#### 1.5.1 支持的Channel
- **需求编号**: FR-CHANNEL-001
- **支持列表**:
  1. **自建Web前端** (`web`)
     - 使用JWT Token认证
     - 支持WebSocket实时通信
     - 支持文件上传下载
  
  2. **钉钉** (`dingtalk`)
     - 支持钉钉机器人
     - 支持单聊和群聊
     - 支持卡片消息展示
  
  3. **飞书** (`feishu`)
     - 支持飞书机器人
     - 支持富文本消息
     - 支持交互式按钮

#### 1.5.2 Channel抽象层
- **需求编号**: FR-CHANNEL-002
- **设计原则**:
  - 统一接口: `BaseChannel`
  - 统一消息格式: `InboundMessage` / `OutboundMessage`
  - 统一认证机制: Channel + User ID
  - 统一文件存储: 按channel和user隔离

### 1.6 存储管理

#### 1.6.1 文件存储结构
```

├── sessions/                          # 会话历史
│   ├── web_user123/                   # web渠道用户user123
│   │   ├── conv_20240315_001.json     # 对话1
│   │   ├── conv_20240315_002.json     # 对话2
│   │   └── metadata.json              # 用户元数据
│   ├── dingtalk_ding456/              # 钉钉用户
│   │   └── conv_20240315_003.json
│   └── feishu_fei789/                 # 飞书用户
│       └── conv_20240315_004.json
├── files/                             # 用户上传文件
│   ├── web_user123/
│   │   ├── data/
│   │   │   └── sales.csv
│   │   └── analysis/
│   │       └── report.py
│   └── dingtalk_ding456/
```

### 1.7 前端功能

#### 1.7.1 对话界面
- **需求编号**: FR-UI-001
- **功能点**:
  - 消息列表展示（用户/助手气泡）
  - 支持Markdown渲染
  - 代码高亮显示
  - 表格数据展示
  - 图片展示
  - 文件上传/下载链接

#### 1.7.2 新建对话
- **需求编号**: FR-UI-002
- **交互流程**:
  1. 点击"新建对话"按钮
  2. 弹出场景选择模态框
  3. 展示场景列表（从后端加载）
  4. 选择场景后点击"创建"
  5. 跳转至新对话页面
  6. 自动聚焦输入框

#### 1.7.3 历史管理
- **需求编号**: FR-UI-003
- **功能点**:
  - 左侧边栏展示历史对话列表
  - 显示对话标题（首条消息摘要）
  - 显示最后更新时间
  - 支持搜索历史对话
  - 支持删除对话
  - 支持导出对话

#### 1.7.4 文件管理
- **需求编号**: FR-UI-004
- **功能点**:
  - 拖拽上传文件
  - 显示已上传文件列表
  - 支持删除已上传文件
  - 支持在对话中引用文件

## 2. 非功能性需求

### 2.1 性能需求

| 指标 | 目标值 | 测试条件 |
|------|--------|----------|
| API响应时间 | P95 < 1s | 单并发 |
| LLM首字时间 | < 2s | 流式响应 |
| SQL执行时间 | < 10s | 查询100万行数据 |
| Python执行 | < 30s | 沙箱执行 |
| 会话加载 | < 500ms | 加载100条消息 |
| 并发用户数 | 1000 | 4核8G服务器 |

### 2.2 安全需求

- **身份认证**: JWT Token + Channel验证
- **权限控制**: 用户级文件系统隔离
- **代码安全**: Python沙箱执行（opensandbox）
- **SQL安全**: 仅允许SELECT查询，自动添加LIMIT
- **文件安全**: 禁止访问系统目录，文件大小限制
- **数据加密**: 敏感数据脱敏展示
- **审计日志**: 记录所有用户操作

### 2.3 可靠性需求

- **可用性**: 99.9%（单机）
- **故障恢复**: 自动重启失败任务
- **数据备份**: 每日备份会话数据
- **容错机制**: LLM调用失败自动重试（3次）

### 2.4 可维护性需求

- **日志级别**: DEBUG/INFO/WARNING/ERROR
- **监控指标**: Prometheus + Grafana
- **健康检查**: HTTP /health 端点
- **配置热更新**: 支持动态加载配置
- **文档完备**: API文档、部署文档、运维手册

### 2.5 可扩展性需求

- **场景扩展**: 通过配置文件添加新场景
- **技能扩展**: 通过注册机制添加新工具
- **模型扩展**: 支持多LLM提供商
- **Channel扩展**: 实现BaseChannel接口添加新渠道

## 3. 数据需求

### 3.1 数据存储需求

| 数据类型 | 存储位置 | 存储格式 | 备份策略 |
|----------|----------|----------|----------|
| 会话历史 | /workspace/sessions/ | JSON | 每日全量 |
| 用户文件 | /workspace/files/ | 原始文件 | 按需备份 |
| 场景配置 | /workspace/shared/ | JSON | 版本控制 |
| 审计日志 | /workspace/logs/ | JSONL | 30天滚动 |

### 3.2 数据保留策略

- **活跃会话**: 永久保留（直到用户删除）
- **归档会话**: 保留1年
- **临时文件**: 7天后自动清理
- **日志文件**: 保留30天
- **审计日志**: 保留90天

## 4. 接口需求

### 4.1 后端API接口

#### 创建对话
```http
POST /api/conversations
{
  "scene_code": "sales_analysis",
  "scene_name": "销售数据分析"
}
```

#### 发送消息
```http
POST /api/conversations/{conversation_id}/messages
{
  "content": "查询上个月的销售额",
  "stream": true
}
```

#### 获取场景列表
```http
GET /api/scenes
```

#### 文件上传
```http
POST /api/files/upload
Content-Type: multipart/form-data
```

详细接口定义见《后端服务架构》文档。

### 4.2 外部服务接口

#### RAG服务接口
```
POST http://my_rag_v1
Content-Type: application/json

{
  "scene_code": "string",
  "query": "string",
  "type": "doc|schema|qa|all"
}
```

#### SQL执行接口
```
POST http://my_hive_sql_exec
Content-Type: application/json

{
  "sql_text": "string",
  "scene_code": "string"
}
```

## 5. 约束和假设

### 5.1 技术约束

- Python 3.10+
- Opensandbox支持（Python沙箱）
- LLM API配额充足
- RAG服务可用性 > 99%
- SQL服务可用性 > 99.9%

### 5.2 业务假设

- 单个会话消息数 < 10000条
- 单个用户文件数 < 1000个
- 单个文件大小 < 50MB
- 并发用户数 < 1000
- 每日新增会话 < 10000

### 5.3 限制条件

- 不支持SQL写入操作（INSERT/UPDATE/DELETE）
- 不支持跨用户数据访问
- 不支持跨会话上下文引用
- 不支持自定义Python库安装
- 不支持长时间运行的后台任务

## 6. 验收标准

### 6.1 功能验收

- [ ] 新建对话：可正常创建对话，生成conversation_id
- [ ] 场景选择：从配置加载场景列表，创建时绑定
- [ ] 对话功能：支持多轮对话，context保持
- [ ] 文件操作：可上传、读取、搜索文件
- [ ] Python执行：可生成和执行Python代码，结果正确
- [ ] RAG查询：正确调用RAG接口并返回结果
- [ ] SQL执行：正确执行查询并返回数据
- [ ] 多用户隔离：用户间数据不可见
- [ ] 会话管理：可查看、搜索、删除历史

### 6.2 性能验收

- [ ] API响应P95 < 1s
- [ ] LLM首字时间 < 2s
- [ ] SQL查询 < 10s
- [ ] Python执行 < 30s
- [ ] 单实例支持1000并发

### 6.3 安全验收

- [ ] 无法访问其他用户文件
- [ ] Python代码在沙箱中执行
- [ ] SQL仅允许SELECT查询
- [ ] 文件大小限制有效
- [ ] JWT认证无法绕过

|------|------|

| Agent | AI智能体，负责消息处理和工具调用 |
| Skill | 工具能力，如文件读取、SQL执行等 |
