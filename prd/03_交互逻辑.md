# ChatBI 交互逻辑设计

## 1. 用户交互概述

ChatBI平台主要面向两类用户：
- **业务分析师**：通过对话查询数据，获取洞察
- **数据工程师**：验证SQL查询，调试Python代码

交互核心目标是**自然、高效、安全**地完成数据分析任务。

## 2. 前端页面设计（简化HTML实现）

### 2.1 技术选型

**简化技术栈**：
- **HTML**: 原生HTML5
- **CSS**: 原生CSS + Flexbox布局
- **JavaScript**: 原生ES6 + Axios（HTTP请求）
- **图标**: Unicode表情符号（无需图标库）
- **UI框架**: 无，使用简单原生组件

**优势**：
- 零依赖，部署简单
- 加载速度快
- 维护成本低
- 兼容性好

### 2.3 核心组件设计（简化版）

### 2.2 页面组件说明

#### 2.2.1 左侧边栏 (Sidebar)

**功能区域**：
1. **新建对话按钮**（常驻顶部）
   - 蓝色主按钮，醒目
   - 点击触发场景选择弹窗

2. **场景分组的历史对话**
   - 按scene_code分组
   - 显示场景名称（折叠/展开）
   - 每个对话显示：标题、最后消息时间、消息数

3. **搜索框**
   - 支持搜索对话内容
   - 实时过滤，高亮匹配文本

4. **设置按钮**
   - 清空所有对话
   - 导出对话记录
   - 主题切换（浅色/深色）

#### 2.2.2 主内容区 (Main Content)

**对话页面**：
1. **对话标题栏**
   - 显示场景名称 + 对话标题（首消息摘要）
   - 右侧：删除对话、分享对话按钮

2. **消息列表**
   - 用户消息：右侧，蓝色气泡
   - 助手消息：左侧，白色气泡
   - 系统消息：居中，灰色提示
   - 支持Markdown渲染
   - 代码块语法高亮
   - 表格滚动展示
   - 图片自适应宽度

3. **输入区域**
   - 文本输入框（支持多行，Shift+Enter换行）
   - 发送按钮（Enter发送）
   - 文件上传按钮（支持拖拽）
   - 正在输入指示（"AI正在思考..."）

### 2.4 对话交互流程

#### 2.4.1 单轮对话时序

```
用户                          前端                        后端
  │                            │                            │
  │ 输入问题                    │                            │
  │───────────────────────────>│                            │
  │                            │                            │
  │                            │ 发送POST /api/messages     │
  │                            │───────────────────────────>│
  │                            │                            │
  │                            │◄─ 返回conversation_id       │
  │                            │                            │
  │  显示"AI正在思考..."       │                            │
  │  用户消息（右侧气泡）       │                            │
  │                            │                            │
  │                            │  GET /api/stream/conv_id   │
  │                            │◄───────────────────────────│
  │                            │                            │
  │                            │  [流式响应]                 │
  │                            │  data: {"content": "根据"}  │
  │  逐步显示消息               │◄───────────────────────────│
  │  "根据查询结果，上月的..."  │  data: {"content": "查询"}  │
  │                            │◄───────────────────────────│
  │                            │  ...                        │
  │                            │                            │
```

#### 2.4.2 多轮对话上下文保持

```
对话1: 查询上个月的销售额
AI: 上个月销售额为1,234,567元

对话2: 按产品类别分解呢？  ← 隐含引用上文
AI: 各产品类别销售如下:
    电子产品: 456,789元
    服装: 345,678元
    ...
```

**技术实现**:
- 前端每次发送conversation_id
- 后端从会话文件加载历史消息
- 构建LLM上下文（滑动窗口，最近N条）

### 2.7 错误处理

#### 2.7.1 用户输入错误

**文件不存在**：
```
❌ 文件不存在: data/abc.csv

```

**SQL语法错误**：
```
❌ SQL执行失败
错误: Syntax error at line 3, column 15
建议: 检查字段名是否正确
```

#### 2.7.2 系统错误

**LLM调用失败**：
```
⚠️ 服务暂时不可用
错误: LLM API调用超时
建议: 请稍后重试，或联系管理员
```

**Python沙箱错误**：
```
❌ Python执行失败
错误: Memory limit exceeded
建议: 减少数据量，或优化代码
```

## 3. 多Channel交互差异

### 3.1 自建Web前端

**优势**：
- 功能最完整
- 支持文件上传/下载
- 支持代码高亮
- 支持图表展示
- 实时流式响应

**交互方式**：
- 文本输入 + 文件上传
- Markdown渲染
- 按钮交互（删除、导出等）

### 3.2 钉钉

**交互限制**：
- 不支持文件上传（仅支持链接）
- Markdown部分支持
- 卡片消息展示结果

**适配方案**：
```
用户: 查询上个月销售额

AI: 查询结果: ¥1,234,567

[查看详情](https://chatbi.com/conv/xxx)
```

**交互方式**：
- @机器人触发
- 纯文本对话
- 链接跳转Web查看详情

### 3.3 飞书

**交互限制**：
- 支持富文本消息
- 支持部分Markdown
- 支持交互式按钮

**适配方案**：
```
┌────────────────────────────────┐
│ 查询结果                       │
├────────────────────────────────┤
│ 销售额: ¥1,234,567            │
│                                │
│ [查看详情] [下载Excel]         │
└────────────────────────────────┘
```

**交互方式**：
- @机器人或私聊
- 卡片消息展示
- 按钮交互（查看详情、下载）

## 4. 用户体验设计原则

### 4.1 即时反馈

- **输入时**: 显示"AI正在输入..."
- **工具调用**: 显示执行进度和状态
- **长时间操作**: 显示预计等待时间
- **错误时**: 明确错误原因和解决建议

### 4.2 渐进式披露

- **初始**: 展示核心功能（对话、文件上传）
- **进阶**: 展示工具调用细节
- **专家**: 展示完整日志、调试信息

### 4.3 容错设计

- **输入容错**: 自动纠正常见错误
- **操作可撤销**: 支持删除消息、回退操作
- **恢复机制**: 断线重连后恢复对话

### 4.4 一致性

- **视觉一致**: 统一的颜色、字体、间距
- **交互一致**: 统一的操作反馈、动画
- **术语一致**: 统一的命名、提示

## 9. 总结

交互设计的核心是**简单、直观、高效**：
- 减少用户认知负担
