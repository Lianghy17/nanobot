# ChatBI 关键流程时序图

## 1. 新建对话流程

### 1.1 完整流程时序图

```mermaid
sequenceDiagram
    participant User as 用户
    participant Frontend as 前端页面
    participant API as ChatBI API
    participant SceneService as 场景服务
    participant ConvService as 对话服务
    participant SessionManager as 会话管理
    participant Storage as 文件存储

    User->>Frontend: 点击"新建对话"按钮
    Frontend->>API: GET /api/scenes
    API->>SceneService: 加载场景配置
    SceneService->>Storage: 读取scenes.json
    Storage-->>SceneService: 返回场景列表
    SceneService-->>API: Scene[]
    API-->>Frontend: 场景列表数据
    Frontend-->>User: 显示场景选择弹窗

    User->>Frontend: 选择场景 + 点击"创建"
    Frontend->>API: POST /api/conversations
    Note over Frontend,API: {scene_code, scene_name}

    API->>ConvService: create_conversation(user_channel, scene)
    ConvService->>ConvService: 生成conversation_id
    ConvService->>SessionManager: 创建会话目录
    SessionManager->>Storage: mkdir -p sessions/{user_channel}/
    Storage-->>SessionManager: 目录创建成功
    SessionManager-->>ConvService: 返回会话路径

    ConvService->>Storage: 写入{conv_id}.json
    Note over ConvService,Storage: 包含初始会话信息
    Storage-->>ConvService: 写入成功
    ConvService-->>API: Conversation对象
    API-->>Frontend: {conversation_id, websocket_url}
    Frontend-->>User: 跳转至对话页面
    Frontend->>User: 自动聚焦输入框
```

### 1.2 时序图说明

**关键步骤**：
1. **场景加载**：从配置文件读取可用场景
2. **会话创建**：生成唯一ID，创建会话文件
3. **目录隔离**：按user_channel创建独立目录
4. **状态初始化**：写入初始会话状态

**异常处理**：
- 场景不存在：返回400错误
- 权限不足：返回403错误
- 存储失败：返回500错误，重试机制

## 2. 对话交互流程

### 2.1 单轮对话（流式响应）

```mermaid
sequenceDiagram
    participant User as 用户
    participant Frontend as 前端
    participant API as ChatBI API
    participant Queue as Loop队列
    participant Agent as Agent引擎
    participant ToolRegistry as 工具注册表
    participant Sandbox as Python沙箱
    participant LLMProvider as LLM服务
    participant RagService as RAG服务
    participant SqlService as SQL服务
    participant Storage as 存储

    User->>Frontend: 输入问题并发送
    Frontend->>API: POST /messages (stream=true)

    API->>Queue: 添加任务到队列
    Note over API,Queue: {conv_id, user_channel, content}
    Queue-->>API: 任务已接收
    API-->>Frontend: 200 OK + SSE流

    Frontend->>Frontend: 显示"AI正在思考..."

    Queue->>Agent: 消费任务
    Agent->>Storage: 加载会话历史
    Storage-->>Agent: 返回messages[]
    Agent->>Agent: 构建LLM上下文

    Agent->>LLMProvider: chat(messages, tools)
    Note over Agent,LLMProvider: 携带可用工具列表
    LLMProvider-->>Agent: 返回tool_calls[rag_search]

    Agent->>Frontend: SSE: {"type": "tool", "tool": "rag_search"}
    Frontend->>User: 显示工具调用状态

    Agent->>ToolRegistry: execute("rag_search", args)
    ToolRegistry->>RagService: POST /my_rag_v1
    Note over ToolRegistry,RagService: {scene_code, query, type}
    RagService-->>ToolRegistry: 返回知识库结果
    ToolRegistry-->>Agent: 工具执行结果

    Agent->>LLMProvider: 再次调用（包含工具结果）
    LLMProvider-->>Agent: 返回tool_calls[execute_sql]

    Agent->>Frontend: SSE: {"type": "tool", "tool": "execute_sql"}
    Agent->>ToolRegistry: execute("execute_sql", args)
    ToolRegistry->>SqlService: POST /my_hive_sql_exec
    SqlService-->>ToolRegistry: 查询结果
    ToolRegistry-->>Agent: 结果

    Agent->>LLMProvider: 最终调用（无工具调用）
    LLMProvider-->>Agent: 最终回复内容

    Agent->>Storage: 保存会话消息
    Storage-->>Agent: 成功

    Agent->>Frontend: SSE: {"type": "message", "content": "..."}
    Frontend->>User: 流式显示AI回复

    Agent->>Frontend: SSE: {"type": "done"}
    Frontend->>Frontend: 关闭SSE连接
```

### 2.2 时序图说明

**关键流程**：
1. **异步处理**：API层将任务放入队列，立即返回SSE流
2. **流式响应**：通过Server-Sent Events实时推送进度
3. **工具链调用**：多个工具按LLM决策顺序执行
4. **状态反馈**：每个工具调用状态实时推送到前端
5. **持久化**：对话完成后保存到存储

**并发控制**：
- 单个用户单会话：顺序执行
- 单个用户多会话：可并行
- 多用户：完全并行

## 3. Python代码执行流程

### 3.1 代码生成与执行

```mermaid
sequenceDiagram
    participant User as 用户
    participant Frontend as 前端
    participant API as API层
    participant Agent as Agent引擎
    participant LLM as LLM服务
    participant ToolRegistry as 工具注册表
    participant Sandbox as Opensandbox
    participant Storage as 文件存储

    User->>Frontend: "帮我分析销售数据并生成图表"
    Frontend->>API: 发送消息
    API->>Agent: 处理请求

    Agent->>LLM: 构建代码生成prompt
    Note over Agent,LLM: 包含数据路径、分析目标
    LLM-->>Agent: 返回Python代码

    Agent->>ToolRegistry: execute("write_python", {path, code})
    ToolRegistry->>Storage: 写入analysis/sales_plot.py
    Storage-->>ToolRegistry: 写入成功
    ToolRegistry-->>Agent: 返回文件路径

    Agent->>Frontend: SSE: "代码已生成..."
    Agent->>ToolRegistry: execute("execute_python", {file_path})

    ToolRegistry->>Sandbox: 创建沙箱环境
    Note over ToolRegistry,Sandbox: user_channel, conv_id
    Sandbox->>Sandbox: 初始化隔离环境
    Sandbox-->>ToolRegistry: 沙箱就绪

    ToolRegistry->>Sandbox: 执行Python文件
    Note over ToolRegistry,Sandbox: 挂载用户目录
    Sandbox->>Storage: 读取数据文件
    Storage-->>Sandbox: 返回CSV数据
    Sandbox->>Sandbox: 执行数据分析
    Sandbox->>Sandbox: 生成图表(sales_chart.png)
    Sandbox->>Storage: 保存图表到用户目录
    Storage-->>Sandbox: 保存成功
    Sandbox-->>ToolRegistry: 执行结果
    Note over Sandbox,ToolRegistry: stdout, stderr, returncode

    ToolRegistry-->>Agent: 返回执行结果
    Agent->>LLM: 构建总结prompt
    LLM-->>Agent: 返回自然语言总结

    Agent->>Frontend: SSE: {"content": "分析完成...", "image_url": "..."}
    Frontend->>User: 显示分析结果和图表
```

### 3.2 时序图说明

**安全机制**：
1. **沙箱隔离**：每个请求独立沙箱
2. **资源限制**：CPU、内存、磁盘、网络
3. **文件访问**：只能访问用户目录下的文件
4. **超时控制**：自动终止超时任务

**错误处理**：
- 代码语法错误：捕获异常，返回错误信息
- 运行时错误：返回stderr，LLM分析原因
- 资源超限：强制终止，返回错误码

## 4. SQL查询流程

### 4.1 SQL生成与执行

```mermaid
sequenceDiagram
    participant User as 用户
    participant Frontend as 前端
    participant API as API层
    participant Agent as Agent引擎
    participant LLM as LLM服务
    participant ToolRegistry as 工具注册表
    participant RagService as RAG服务
    participant SqlService as SQL执行服务
    participant Storage as 会话存储

    User->>Frontend: "查询最近7天的日销售额"
    Frontend->>API: 发送消息
    API->>Agent: 处理请求

    Agent->>Storage: 加载会话历史
    Storage-->>Agent: 返回场景编码

    Agent->>ToolRegistry: execute("rag_search", args)
    Note over Agent,ToolRegistry: 查询表结构和示例
    ToolRegistry->>RagService: POST /my_rag_v1
    Note over ToolRegistry,RagService: {scene_code, query, type}
    RagService-->>ToolRegistry: 返回schema和QA示例
    ToolRegistry-->>Agent: 知识库结果

    Agent->>LLM: 构建SQL生成prompt
    Note over Agent,LLM: 包含表结构、字段、示例SQL
    LLM-->>Agent: 返回SQL语句
    Note over Agent,LLM: SELECT date, SUM(amount) FROM sales...

    Agent->>ToolRegistry: execute("execute_sql", {sql})
    ToolRegistry->>SqlService: POST /my_hive_sql_exec
    Note over ToolRegistry,SqlService: {sql_text, scene_code}

    SqlService->>SqlService: SQL安全检查
    Note over SqlService: - 禁止DDL/DML<br/>- 添加LIMIT<br/>- 检查表权限
    SqlService->>Hive: 执行查询
    Hive-->>SqlService: 返回结果集
    SqlService-->>ToolRegistry: {data, columns, row_count}

    ToolRegistry-->>Agent: 查询结果
    Agent->>LLM: 构建总结prompt
    Note over Agent,LLM: 包含查询结果
    LLM-->>Agent: 返回自然语言总结

    Agent->>Storage: 保存会话消息
    Agent->>Frontend: SSE: 流式返回总结
    Frontend->>User: 显示查询结果
```

### 4.2 SQL安全检查

```mermaid
flowchart TD
    Start([接收到SQL]) --> Parse[解析SQL语句]
    Parse --> CheckType{检查语句类型}
    
    CheckType -->|DDL| Reject[❌ 拒绝执行<br/>CREATE/DROP/ALTER]
    CheckType -->|DML| Reject2[❌ 拒绝执行<br/>INSERT/UPDATE/DELETE]
    CheckType -->|DQL| CheckTable[检查表权限]
    
    CheckTable --> Allowed{表在<br/>允许列表?}
    Allowed -->|否| Reject3[❌ 表无权限]
    Allowed -->|是| AddLimit[自动添加LIMIT]
    
    AddLimit --> CheckLimit{已有LIMIT?}
    CheckLimit -->|否| Add[添加 LIMIT 10000]
    CheckLimit -->|是| CheckSize{LIMIT值过大?}
    
    CheckSize -->|是| Replace[替换为 LIMIT 10000]
    CheckSize -->|否| Pass[✅ 通过检查]
    Add --> Pass
    Replace --> Pass
    
    Pass --> Execute[执行SQL查询]
    Reject --> Error[返回错误]
    Reject2 --> Error
    Reject3 --> Error
```

## 6. 错误处理和恢复流程

### 6.1 LLM调用失败重试

```mermaid
sequenceDiagram
    participant Agent as Agent引擎
    participant LLM as LLM服务
    participant Queue as 重试队列
    participant Alert as 告警系统

    Agent->>LLM: 调用LLM API
    LLM-->>Agent: ❌ 超时/错误
    
    Agent->>Queue: 记录重试次数 (retry=1)
    Note over Agent,Queue: 延迟2秒后重试
    
    Queue->>Agent: 重试任务
    Agent->>LLM: 再次调用LLM API
    LLM-->>Agent: ❌ 仍然失败
    
    Agent->>Queue: 记录重试次数 (retry=2)
    Note over Agent,Queue: 延迟4秒后重试
    
    Queue->>Agent: 重试任务
    Agent->>LLM: 第三次调用
    LLM-->>Agent: ❌ 失败
    
    Agent->>Alert: 发送告警通知
    Note over Agent,Alert: LLM服务异常
    Alert-->>Agent: 告警已发送
    
    Agent->>Agent: 返回错误消息给用户
    Note over Agent: "抱歉，服务暂时不可用，请稍后重试"
```

### 6.2 沙箱执行失败处理

```mermaid
flowchart TD
    Start([Python执行]) --> Execute[执行代码]
    Execute --> Success{成功?}
    
    Success -->|是| Return[返回结果]
    Success -->|否| ErrorType{错误类型}
    
    ErrorType -->|SyntaxError| LLMFix[让LLM修复语法]
    ErrorType -->|ImportError| LLMFix2[让LLM替换库]
    ErrorType -->|Timeout| Timeout[返回超时错误]
    ErrorType -->|MemoryError| Resource[返回资源限制错误]
    ErrorType -->|PermissionError| Security[返回安全限制错误]
    
    LLMFix --> Regenerate[重新生成代码]
    LLMFix2 --> Regenerate
    
    Regenerate --> ReExecute[重新执行]
    ReExecute --> Success2{成功?}
    
    Success2 -->|是| Return
    Success2 -->|否| GiveUp[放弃执行<br/>返回错误]
    
    Timeout --> User[提示用户<br/>优化代码]
    Resource --> User
    Security --> User
```

## 8. 会话持久化和恢复

### 8.1 会话保存流程

```mermaid
sequenceDiagram
    participant Agent as Agent引擎
    participant Session as 会话管理
    participant Storage as 存储
    participant Backup as 备份服务

    loop 每5秒或每5条消息
        Agent->>Session: 保存会话状态
        Session->>Session: 序列化会话数据
        Session->>Storage: 写入临时文件
        Note over Session,Storage: {conv_id}.json.tmp
        Storage-->>Session: 写入成功
        
        Session->>Storage: 重命名为正式文件
        Note over Session,Storage: {conv_id}.json
        Storage-->>Session: 成功
    end
    
    loop 每天凌晨2点
        Storage->>Backup: 触发备份
        Backup->>Storage: 扫描会话文件
        Storage-->>Backup: 文件列表
        
        Backup->>Storage: 读取文件
        Storage-->>Backup: 文件内容
        Backup->>Backup: 压缩打包
        Backup->>Backup: 上传至备份存储
        Note over Backup: S3/OSS/NFS
    end
    
    alt 服务重启
        Agent->>Session: 恢复会话
        Session->>Storage: 加载所有会话文件
        Storage-->>Session: 会话数据
        Session->>Session: 重建会话对象
        Session-->>Agent: 会话恢复完成
    end
```

### 8.2 会话恢复

```mermaid
sequenceDiagram
    participant User as 用户
    participant Frontend as 前端
    participant API as API层
    participant Session as 会话管理
    participant Storage as 存储

    User->>Frontend: 打开历史对话
    Frontend->>API: GET /conversations/{conv_id}
    
    API->>Session: load_session(conv_id, user_channel)
    Session->>Storage: 读取{conv_id}.json
    Storage-->>Session: 返回会话数据
    
    alt 文件不存在
        Storage-->>Session: FileNotFound
        Session-->>API: 会话不存在错误
        API-->>Frontend: 404 Not Found
        Frontend-->>User: 显示"会话不存在"
    else 文件损坏
        Storage-->>Session: JSONDecodeError
        Session->>Backup: 尝试从备份恢复
        Backup-->>Session: 备份数据
        Session->>Storage: 写入修复后的文件
        Storage-->>Session: 成功
        Session-->>API: 返回会话数据
    else 正常
        Session-->>API: Conversation对象
        API->>API: 加载消息历史
        API-->>Frontend: {messages[], scene_info}
        Frontend->>Frontend: 渲染消息列表
        Frontend-->>User: 显示历史对话
    end
```

### 10.1 LLM调用性能优化

```mermaid
sequenceDiagram
    participant Agent as Agent引擎
    participant Cache as 缓存层
    participant LLMProvider as LLM提供商
    participant Pool as 连接池

    Agent->>Cache: 查询缓存<br/>get(messages_hash)
    
    alt 缓存命中
        Cache-->>Agent: 返回缓存结果
        Note over Agent: 跳过LLM调用
    else 缓存未命中
        Cache-->>Agent: 未找到
        
        Agent->>Pool: 获取HTTP连接
        Pool-->>Agent: 返回连接
        
        Agent->>LLMProvider: 发送请求
        Note over Agent,LLMProvider: 复用连接
        
        LLMProvider-->>Agent: 返回响应
        
        Agent->>Cache: 写入缓存<br/>set(messages_hash, result)
        Note over Agent,Cache: TTL: 3600秒
    end
    
    alt 流式响应
        LLMProvider-->>Agent: [chunk1]
        Agent->>Agent: 处理chunk
        LLMProvider-->>Agent: [chunk2]
        Agent->>Agent: 处理chunk
        LLMProvider-->>Agent: [done]
    else 非流式
        LLMProvider-->>Agent: 完整响应
    end
```

## 时序图使用说明

### 如何阅读时序图

1. **时间顺序**：从上到下，表示时间流逝
2. **参与者**：左侧是系统组件，右侧是用户或外部服务
3. **消息箭头**：
   - 实线箭头：同步调用，等待响应
   - 虚线箭头：异步调用或返回
   - 平行线：并行执行
4. **Note**：注释框用于说明关键逻辑或条件判断

### 关键路径标识

- **红色路径**：错误处理路径
- **绿色路径**：主流程成功路径
- **蓝色路径**：优化或缓存路径
- **黄色区域**：并行执行区域

### 性能关键点

- **⭐**：性能优化点
- **⚠️**：潜在风险点
- **🔒**：安全控制点
- **💾**：持久化点
